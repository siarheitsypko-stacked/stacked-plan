🗺️ XML Mind Map 🧩

Карточная схема ключевых концепций, синтаксиса и экосистемы XML — от основ до современных инструментов

📄🌐

Введение в XML

eXtensible Markup Language — гибкий язык разметки для хранения и передачи данных, независимый от платформы. 
<?xml version="1.0" encoding="UTF-8"?>
<bookstore>
  <book category="fiction">
    <title>XML Basics</title>
    <author>J. Doe</author>
  </book>
</bookstore>
📌 Пролог + корневой элемент <bookstore>. XML сохраняет структуру данных.
⚙️📐

Синтаксис XML

Well‑formed (правильно сформированный) документ: закрывающие теги, регистрозависимость, атрибуты в кавычках. 
<!-- комментарий -->
<person age="30">
  <name>Alice</name>
  <![CDATA[ <special> raw <data> ]]>
</person>
<empty />   <!-- пустой элемент -->
🏷️ Имена чувствительны к регистру, атрибуты только в кавычках. CDATA сохраняет спецсимволы.
📜🔖

DTD

Определение типа документа. Внутренние / внешние DTD. Описывает структуру и допустимые элементы. 
<!DOCTYPE note [
  <!ELEMENT note (to,from,body)>
  <!ELEMENT to (#PCDATA)>
  <!ATTLIST note type CDATA #IMPLIED>
]>
<note type="reminder">
  <to>Alex</to><from>Jane</from><body>Hi!</body>
</note>
🔁 DTD позволяет валидировать legacy-системы, но уступает XSD в строгости типов.
📐🧾

XML Schema (XSD)

Строгая валидация, пространства имён, типы данных и ограничения (minOccurs, pattern). 
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
  <xs:element name="product">
    <xs:complexType>
      <xs:sequence>
        <xs:element name="code" type="xs:string"/>
        <xs:element name="price" type="xs:decimal"/>
      </xs:sequence>
      <xs:attribute name="id" type="xs:integer" use="required"/>
    </xs:complexType>
  </xs:element>
</xs:schema>
📦 XSD поддерживает пространства имён, наследование и ограничения (enumeration, pattern).
🧭🔍

XPath & XQuery

XPath — язык навигации по XML-дереву. XQuery — мощный язык запросов, похожий на SQL для XML. 
XPath примеры:
/bookstore/book[price>30]/title
//@category

XQuery:
for $b in doc("books.xml")/bookstore/book
where $b/price < 40
return $b/title
📍 XPath предикаты фильтруют узлы. XQuery используется в базах данных (eXist, MarkLogic).
🔄🎨

XSLT

Преобразование XML в HTML, текст или другой XML. Шаблоны и правила выборки. 
<xsl:stylesheet version="1.0">
  <xsl:template match="/">
    <html><body>
      <xsl:for-each select="catalog/book">
        <p><xsl:value-of select="title"/></p>
      </xsl:for-each>
    </body></html>
  </xsl:template>
</xsl:stylesheet>
⚡ XSLT 3.0 поддерживает потоковую обработку и создание сложных отчётов.
🏷️🌍

Пространства имён

xmlns:prefix="URI" — позволяют смешивать разные словари (XML Schema + SVG + XHTML). 
<root xmlns:h="http://www.w3.org/1999/xhtml"
      xmlns:svg="http://www.w3.org/2000/svg">
  <h:p>текст в HTML</h:p>
  <svg:circle r="10" />
</root>
🔗 Префиксы связывают элементы с уникальными URI. Предотвращают конфликты имён.
💻⚙️

Обработка XML

DOM (память целиком), SAX (событийный поток), StAX (pull-парсинг). Примеры для разных языков. 
// JavaScript (DOM)
const parser = new DOMParser();
const xmlDoc = parser.parseFromString(xmlString, "text/xml");
let title = xmlDoc.getElementsByTagName("title")[0].textContent;

# Python (ElementTree)
import xml.etree.ElementTree as ET
tree = ET.parse('data.xml')
root = tree.getroot()
🧩 SAX — низкое потребление памяти, DOM — удобная навигация, StAX — баланс.
🔗📡

Связанные технологии

XML-RPC / SOAP (веб-сервисы), RSS/Atom (ленты), SVG, Office Open XML, OpenDocument. 
<!-- пример SVG как XML-приложение -->
<svg width="100" height="100" xmlns="http://www.w3.org/2000/svg">
  <circle cx="50" cy="50" r="40" fill="teal"/>
</svg>

<!-- RSS фрагмент -->
<rss version="2.0">
  <channel><title>XML News</title></channel>
</rss>
🌐 SOAP, SVG, офисные форматы (docx) — повсеместно используют XML-базу.
🛠️🧰

Инструменты & экосистема

Редакторы: XMLSpy, Oxygen, VS Code. Парсеры: libxml2, Xerces, JAXP. XSLT-процессоры и валидаторы. 
# CLI валидация через xmllint
xmllint --noout --schema schema.xsd data.xml

# форматирование XML
xmllint --format ugly.xml > pretty.xml

# популярные плагины VS Code: Red Hat XML, XSLT/XPath
⚙️ XML инструменты обеспечивают автодополнение, отладку XSLT и визуальное редактирование.
