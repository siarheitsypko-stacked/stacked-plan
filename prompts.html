📚 Обучение C# / .NET

Объясни как новичку:
"Объясни разницу между абстрактным классом и интерфейсом в C# с примерами кода. Уровень: начинающий."
Генерация учебных примеров:
"Создай простую консольную программу на C#, которая демонстрирует использование LINQ для фильтрации списка объектов. Добавь комментарии."
Разбор ошибки:
"Вот код (вставка). Компилятор выдаёт ошибку CS1061. Объясни причину и предложи исправление."
Сравнение подходов:
"Сравни подходы Clean Architecture и N-tier для ASP.NET Core. В каких случаях какой лучше?"

⚙️ Генерация и рефакторинг кода

Создание контроллера API:
"Сгенерируй ASP.NET Core контроллер для сущности Post со стандартными CRUD-методами, используя Entity Framework Core. Используй DTO."
Рефакторинг:
"Вот метод (вставка). Как его можно улучшить, следуя принципам чистого кода и SOLID?"
Unit-тесты:
"Напиши xUnit тесты для следующего метода (вставка). Используй Moq для мокирования зависимостей."
Конвертация в TypeScript:
"Перепиши этот JavaScript код на TypeScript с правильной типизацией."

🐞 Отладка и объяснение ошибок

Анализ стека ошибок:
"Вот полный стек ошибки из приложения ASP.NET Core. Определи возможную причину и предложи пошаговое решение."
Почему не работает:
"Мой код не сохраняет данные в базу. Вот модель, контекст и контроллер. Что я упустил?"
Оптимизация производительности:
"Этот LINQ-запрос выполняется медленно на большом наборе данных. Как его оптимизировать?"

🏗️ Архитектура и проектирование

Проектирование API:
"Спроектируй RESTful API для блога с постами, категориями и комментариями. Напиши примеры эндпоинтов и структуру ответов."
Выбор паттерна:
"В моём проекте нужно отправлять уведомления после создания поста. Какой паттерн (Mediator, Observer, Events) лучше использовать и почему?"
Документирование архитектуры:
"Помоги описать архитектуру моего проекта (Clean Architecture + CQRS) для технической документации. Сделай краткий обзор слоёв."

📝 Технический английский

Проверка коммита:
"Проверь сообщение коммита на английском: 'fixed bug in validation'. Исправь, если нужно, в соответствии с Conventional Commits."
Перевод технического текста:
"Переведи следующий текст на технически грамотный английский (для документации): (вставка текста на русском)."
Описание проекта:
"Помоги написать краткое описание моего проекта Stacked на английском для LinkedIn и GitHub."
