📋 Техническое задание

Stacked — двуязычный блог-портфолио разработчика
Версия: 1.0 | Дата: 23 марта 2026

1. Цель и назначение проекта

Создать полноценный веб-сайт (блог-портфолио), который будет:

  • Демонстрировать профессиональные навыки в области C#, .NET, React, TypeScript.
  • Служить платформой для публикации технических статей (на русском и английском).
  • Автоматически отображать репозитории с GitHub (звёзды, форки, описание).
  • Включать AI-ассистента (чат) для интерактивной помощи по коду и статьям.
  • Интегрироваться с Telegram‑ботом для управления контентом и вопросов.
  • Быть основой портфолио при поиске работы, фрилансе и релокации.

2. Технологический стек

  • Бэкенд: C# / .NET 8/9, ASP.NET Core Web API, Entity Framework Core, PostgreSQL (или SQLite для разработки).
  • Фронтенд: React 18+, TypeScript, React Router, Redux Toolkit, Tailwind CSS (или CSS Modules).
  • Инфраструктура: Docker, docker-compose, GitHub Actions (CI/CD), Azure (App Services, SQL, Blob Storage).
  • Интеграции: GitHub API (Octokit), Telegram Bot API, локальный AI (DeepSeek-R1 через Ollama), SignalR (для чата).
  • Тестирование: xUnit, Moq, React Testing Library, Cypress/Playwright (опционально).

3. Функциональные требования

3.1. Публичная часть (блог)

  • Главная страница со списком последних статей (пагинация, по 10 статей).
  • Страница отдельной статьи (заголовок, текст, дата, теги, комментарии).
  • Страницы категорий и тегов.
  • Страница «О себе» (биография, контакты, ссылки на GitHub/LinkedIn).
  • Страница проектов с GitHub (автоматическая подгрузка репозиториев, фильтрация по языку).
  • Переключение языков (русский/английский) с сохранением выбора в localStorage.
  • SEO-оптимизация (метатеги, sitemap.xml, robots.txt).

3.2. Административная часть

  • Защищённый вход (JWT, роли: admin/reader).
  • CRUD для статей (WYSIWYG-редактор или Markdown).
  • Управление категориями и тегами.
  • Модерация комментариев.
  • Обновление данных из GitHub (ручное или по расписанию).
  • Просмотр статистики (просмотры, комментарии).
  • Встроенный AI-чат для помощи (отвечает на вопросы по коду/статьям).

3.3. Интеграция с GitHub

  • Авторизация через OAuth (опционально).
  • Фоновый сервис (IHostedService) для синхронизации репозиториев (раз в сутки).
  • Хранение данных репозиториев в локальной БД (название, описание, язык, звёзды, форки, дата последнего коммита).
  • Отображение README.md на странице проекта.

3.4. AI-ассистент и Telegram‑бот

  • Локальный AI-сервер (Ollama + DeepSeek-R1 1.5b / 7b).
  • Чат в админке: отправка запросов к AI, вывод ответа, сохранение истории (опционально).
  • Telegram‑бот: команды /start, /help, /ask <вопрос> (через AI), /newpost (создание поста).
  • Бот должен работать на том же сервере, что и API (контейнеризация).

3.5. Производительность и безопасность

  • Кэширование популярных страниц (Redis или in-memory).
  • Защита от XSS, CSRF, SQL-инъекций.
  • Валидация всех входных данных (FluentValidation).
  • JWT с коротким временем жизни, refresh-токены.
  • Поддержка HTTPS (Let's Encrypt).

4. Нефункциональные требования

  • Масштабируемость: архитектура построена на принципах Clean Architecture, слои разделены.
  • Тестируемость: юнит-тесты покрывают ключевую бизнес-логику (≥70%).
  • Документация: Swagger для API, README.md, комментарии в коде (XML).
  • Деплой: автоматический через GitHub Actions (push в main).
  • Доступность: сайт должен работать 24/7, минимальное время отклика < 2 секунд.

5. Этапы разработки (согласно роадмапу)

  • Этап 1 (месяц 1): Консольный прототип BlogConsole (CRUD, файловое хранилище).
  • Этап 2 (месяц 1–2): Web API (ASP.NET Core, EF Core, SQLite/PostgreSQL, аутентификация).
  • Этап 3 (месяц 2): Фронтенд на React + TypeScript (компоненты, роутинг, Redux).
  • Этап 4 (месяц 3): Интеграция с GitHub API, AI-ассистент, Telegram-бот.
  • Этап 5 (месяц 4): Docker, CI/CD, деплой в Azure, фриланс-подготовка.

Детальный план каждого дня описан в detailed-plan.html.

6. Критерии успешной сдачи

  • Все функциональные требования реализованы и протестированы.
  • Код выложен на GitHub, имеется история коммитов (английские сообщения).
  • Сайт доступен по постоянной ссылке (GitHub Pages или Azure).
  • Проект описан в профиле разработчика (profile.html).
  • Проведено не менее трёх тестовых запусков с реальными пользователями (друзья/коллеги).
  • Подготовлена презентация проекта (видео или PDF) для работодателя.
Техническое задание является живым документом и может уточняться в процессе разработки.
